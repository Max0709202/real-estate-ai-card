# AIエージェント回答精度改善 — 対応状況のご報告

作成日：2026年6月20日

---

## 1. 概要

ご依頼いただいた「推測回答・誤回答・出典不一致の防止」について、改善を実施しました。

**「回答率より正答率を優先」「出典整合性は必須」「推測回答は禁止」** という基本方針に沿って、システムを2段構えで設計しています。

- **第1層（システム側）：** データが取得できない項目は、AIに渡る前にシステム側で遮断
- **第2層（AI側）：** 回答前の確認ロジック（confidence判定）と回答ポリシーを組み込み

---

## 2. ご依頼項目への対応結果

| ご依頼内容 | 対応状況 |
|---|---|
| ① データにない情報を作らない（学校名・住所・ハザード・災害履歴の推測禁止） | ✅ 対応済 |
| ② 出典表示ルール（実際に取得したデータの出典のみ表示） | ✅ 対応済 |
| ③ 学区回答の制御（住所確定＋学区データがある場合のみ回答） | ✅ 対応済 |
| ④ マンション名が曖昧な場合の候補提示（勝手に1件選ばない） | ✅ 対応済 |
| ⑤「条件整理を進めてください」を希望エリアとして保存しない | ✅ 対応済 |
| ⑥ ハザード情報の回答制御（データなき場合は一般論で答えない） | ✅ 対応済 |
| ⑦ 災害履歴の回答制御（根拠データがある場合のみ） | ✅ 対応済 |
| ⑧ 誤字・意味不明な入力への確認対応 | ⚠️ 一部対応（下記参照） |

## 3. 仕組みのポイント

- **データが取得できない項目は、AIに渡る前にシステム側で遮断**しています。
  確認できない内容は「現在のデータでは確認できませんでした。正確な確認には、自治体公式情報または該当データベースでの確認が必要です」と返す設計です。

- **出典は実際に使ったデータソースのみ表示**します。
  例えば全国マンションデータベースを、学区・ハザード・災害履歴の出典として表示することはありません。

- マンション名が複数候補ある場合は、**AIが勝手に1件を選ばず候補を提示**し、
  お客様に物件を特定していただいてから次の処理へ進みます。

- **回答前のconfidence判定**で、以下を確認してから回答します。
  - 物件または住所が一意に特定できているか
  - 回答に必要なデータソースが実際に提示されているか
  - 出典を正しく表示できるか

  いずれかが不足する場合は、断定回答を行いません。

---

## 4. 残課題（⑧について）

誤字・意味不明な入力への「聞き返し」は、通常のチャット会話では機能します。

ただし「条件整理」モードでの項目入力時に、漢字を含む誤字（例：「地図wlひぃ」）が
希望エリアとして登録されてしまうケースが残っています。

こちらは追加対応で塞ぐことが可能です。**実施可否をご確認ください。**

---

## 5. 設計目標

不動産購入支援では、誤情報の回答が大きな信用低下につながるため、
**「回答率70％でも正答率99％」** を目指す設計方針で構築しております。

- 回答率：下がってもよい
- 正答率：最優先
- 出典整合性：必須
- 推測回答：禁止
