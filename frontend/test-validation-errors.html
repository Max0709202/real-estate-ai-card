Validation Error Display Test

This page demonstrates the improved error handling that displays all validation errors from the API response.

Test 1: Single Error

Scenario: Only one field has a validation error

Expected Result: Display the general message and the single field error

{
  "success": false,
  "message": "入力内容に誤りがあります",
  "errors": {
    "email": "有効なメールアドレスを入力してください"
  }
}

Test 2: Multiple Errors

Scenario: Multiple fields have validation errors (email, password, phone)

Expected Result: Display all field errors in a list

{
  "success": false,
  "message": "入力内容に誤りがあります",
  "errors": {
    "email": "有効なメールアドレスを入力してください",
    "password": "パスワードは8文字以上で入力してください",
    "phone_number": "有効な電話番号を入力してください"
  }
}

Test 3: All Field Errors

Scenario: All fields have validation errors

Expected Result: Display all four field errors

{
  "success": false,
  "message": "入力内容に誤りがあります",
  "errors": {
    "email": "有効なメールアドレスを入力してください",
    "password": "パスワードは8文字以上で入力してください",
    "phone_number": "有効な電話番号を入力してください",
    "user_type": "ユーザータイプを選択してください"
  }
}

Test 4: No Field Errors (General Error Only)

Scenario: General error without specific field errors

Expected Result: Display only the general message

{
  "success": false,
  "message": "このメールアドレスは既に登録されています"
}
